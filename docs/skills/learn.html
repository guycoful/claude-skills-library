מה הסקיל הזה עושה?

הסקיל מאפשר לקלוד ללמוד ספריות וטכנולוגיות חדשות אוטומטית מהדוקומנטציה שלהן, ולהפוך את הידע הזה לסקיל שימושי שנשמר לשימוש עתידי.

1 איך זה עובד?

כשתבקשו מקלוד ללמוד טכנולוגיה חדשה, הוא:

  1. מחפש את הדוקומנטציה הרשמית

    משתמש ב-Context7 MCP או בחיפוש אינטרנט למציאת הדוקס

  2. מחלץ את המידע החשוב

    התקנה, קונספטים, דפוסי שימוש נפוצים, API ומלכודות

  3. מוודא קונבנציות

    מתייעץ עם claude-code-guide לגבי הפורמט הנכון לכתיבת סקילים

  4. יוצר סקיל מובנה

    שומר את הידע בפורמט סקיל שקל להשתמש בו

🧠

מטא-סקיל: זהו סקיל שיוצר סקילים אחרים! הוא מרחיב את היכולות של קלוד קוד באופן אוטומטי.

2 איך להשתמש?

פשוט תבקשו מקלוד ללמוד משהו חדש:

# יצירת סקיל חדש /learn stripe-payments # עדכון סקיל קיים /learn stripe-payments --update # שמירה בפרויקט במקום גלובלי /learn supabase --project

דוגמאות לשימוש

# ספריות JavaScript "למד את react-query" "/learn tanstack-router" # APIs ושירותים "learn about openai api" "/learn twilio" # כלי פיתוח "צור לי סקיל לעבודה עם docker" "/learn kubernetes"

פרמטרים

פרמטר תיאור דוגמה
<topic> שם הספרייה/טכנולוגיה /learn stripe
--update עדכון סקיל קיים /learn stripe --update
--project שמירה בפרויקט הנוכחי /learn prisma --project

4 דרישות (אופציונלי)

הסקיל עובד גם בלי הגדרות נוספות, אבל עובד טוב יותר עם:

Context7 MCP Plugin (מומלץ)

מספק גישה ישירה לדוקומנטציה של אלפי ספריות. להתקנה:

# הוסיפו לקובץ ההגדרות של Claude Code # ~/.claude/settings.json או .claude/settings.json { "mcpServers": { "context7": { "command": "npx", "args": ["-y", "@context7/mcp"] } } }
💡

בלי Context7: הסקיל ישתמש ב-WebSearch כגיבוי ויחפש דוקומנטציה באינטרנט.

טיפים לשימוש מיטבי

בחירת נושאים טובים

  • ספריות עם דוקומנטציה טובה (Stripe, Supabase, etc.)
  • APIs עם reference ברור
  • כלים שמשתמשים בהם הרבה

מתי לעדכן?

  • כשיוצא major version חדש
  • כשמגלים שחסר משהו בסקיל
  • כשהדוקומנטציה משתנה משמעותית
⚠️

חשוב: הסקיל לא מעתיק דוקומנטציה מילה במילה (זכויות יוצרים). הוא מחלץ את המידע החשוב ומארגן אותו מחדש.
