1 התקנה

התקנת תלויות

ראשית, ודאו שכל 6 הסקילים התלויים מותקנים. תוכלו להוריד אותם מהעמוד הראשי.

התקנת הסקיל

# יצירת תיקיית הסקיל mkdir -p ~/.claude/skills/tutorial-creator # חילוץ הקובץ שהורדתם unzip tutorial-creator.zip -d ~/.claude/skills/tutorial-creator

בדיקת מבנה

~/.claude/skills/tutorial-creator/ └── SKILL.md # הנחיות הסקיל

2 הגדרת API Keys

ודאו שקובצי .env בכל הסקילים התלויים מכילים את המפתחות הנדרשים:

Gemini API (לניתוח וידאו)

# ~/.claude/skills/image-generation/scripts/.env GEMINI_API_KEY=your_gemini_api_key

ElevenLabs API (לאודיו)

# ~/.claude/skills/speech-generator/scripts/.env ELEVENLABS_API_KEY=your_elevenlabs_api_key
💡

אותו מפתח ElevenLabs משמש ל-speech-generator, transcribe ו-music-generator

3 שימוש

דרך קלוד קוד

פשוט בקשו מקלוד ליצור טוטוריאל:

# אפשרות 1: עם נתיב לקובץ "תיצור טוטוריאל מהקלטה ~/recordings/demo.mp4" # אפשרות 2: תיאור כללי "יש לי הקלטת מסך שמדגימה את פיצ'ר המשימות של קלוד קוד. תהפוך אותה לטוטוריאל מקצועי"

מה קלוד ישאל אתכם

  • מה הפיצ'ר/נושא שמודגם?
  • מה המסר המרכזי?
  • אורך רצוי: קצר (30-45s) / בינוני (60-90s) / ארוך (2-3 דקות)

מה יקרה אחר כך

  • כתיבת תסריט בסגנון שלכם
  • ניתוח הוידאו עם Gemini למציאת נקודות מפתח
  • יצירת קריינות + מוזיקת רקע
  • עריכת הוידאו (האטה/האצה, מיקס אודיו, כתוביות)
  • שליחה לווטסאפ לבדיקה
  • העלאה ליוטיוב (unlisted)
  • הצעה לפוסט שיווקי (דורש אישור)

4 Script-First Workflow

הסקיל משתמש בגישת Script-First - קודם כותבים את התסריט, ואז מחפשים נקודות בוידאו שמתאימות לו:

# הגישה הרגילה (פחות טובה): וידאו → ניתוח → תסריט → אודיו # הגישה של הסקיל הזה (טובה יותר): קלט משתמש → תסריט → ניתוח וידאו לפי רמזים מהתסריט → אודיו
💡

למה זה עובד יותר טוב? התסריט מגדיר מה חשוב, לא הוידאו. ניתוח הוידאו מכוון (חיפוש רמזים ספציפיים). סנכרון טוב יותר בין קריינות לוידאו.
